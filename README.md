# BluePrint3 website

The BluePrint3 one-page marketing site — an Astro build of `templates/website/Website.dc.html`
from the **BluePrint3 Design System** project on claude.ai/design.

Hosted on Cloudflare. The page itself is prerendered static HTML; the only
server-side route is the quote form endpoint, which runs as a Cloudflare Worker
function and emails enquiries via [Resend](https://resend.com).

## Commands

| Command | Action |
| :-- | :-- |
| `npm install` | Install dependencies |
| `npm run dev` | Dev server at `localhost:4321` — **run this in a real terminal**, see below |
| `npm run check` | Typecheck `.astro` / `.ts` |
| `npm run build` | Build to `./dist/` (`client/` = static assets, `server/` = the Worker) |
| `npm run preview` | Build, then serve it in the real Cloudflare runtime via Wrangler |
| `npm run deploy` | Build and deploy to Cloudflare |
| `npm run og` | Regenerate `public/og.png` (the link-preview card) |
| `npm run types` | Regenerate `worker-configuration.d.ts` after editing `wrangler.jsonc` |

### ⚠️ Don't run the dev server in background mode

`astro dev --background` (and any non-TTY invocation) **makes every request return
500**, including ones that rendered perfectly. It's an upstream bug, not your code:
background mode selects Astro's JSON logger, which writes via `process.stdout`, but
the Cloudflare adapter runs the request handler inside `workerd` where `process`
doesn't exist. The logger throws while logging the request and the 500 it produces
(`process is not defined`) replaces the real response.

Use `npm run dev` in a terminal, or `npm run preview` for a headless check.

## The quote form

`src/pages/api/quote.ts` is the only route with `prerender = false`. It:

- rejects cross-origin posts (Astro's built-in CSRF origin check),
- silently absorbs bots that fill the hidden `company` honeypot,
- validates name / email / message and trims every field to a maximum length,
- sends the enquiry through Resend with `reply_to` set to the enquirer, so you
  can just hit reply.

It deliberately **fails loudly** if it isn't configured — a quote form that
accepts an enquiry and drops it is worse than one that admits it's broken.

Progressive enhancement: with JS the form posts in the background and swaps in the
"Got it." panel. Without JS it posts natively and the endpoint redirects to
`/quote-sent` or `/quote-error`.

### Configuration

Three secrets are required. Locally, copy `.dev.vars.example` to `.dev.vars`
(gitignored) and fill it in:

| Variable | Meaning |
| :-- | :-- |
| `RESEND_API_KEY` | From <https://resend.com/api-keys> |
| `QUOTE_TO_EMAIL` | Where enquiries are delivered |
| `QUOTE_FROM_EMAIL` | Sender — must be on a domain verified at <https://resend.com/domains> |

In production, set them as secrets (never in `wrangler.jsonc`):

```sh
npx wrangler secret put RESEND_API_KEY
npx wrangler secret put QUOTE_TO_EMAIL
npx wrangler secret put QUOTE_FROM_EMAIL
```

Until they're set, the form returns "The quote form isn't set up yet."

## Deploying

> **Note:** this targets **Cloudflare Workers**, not Cloudflare Pages. The Astro
> Cloudflare adapter [dropped Pages support](https://docs.astro.build/en/guides/integrations-guide/cloudflare/);
> Workers-with-static-assets is its replacement and lives in the same
> "Workers & Pages" section of the dashboard, with the same Git-connected builds.

```sh
npm run deploy
```

Or connect the repo in the Cloudflare dashboard with build command `npm run build`.
Set the three secrets above in the dashboard under **Settings → Variables and Secrets**.

Then update `site` in `astro.config.mjs` to the real domain — it's what the
canonical tag, `og:url`, `robots.txt` and the sitemap are built from.

## Structure

```text
src/
├── components/     # the design system, ported 1:1 from the Claude Design project
├── layouts/
│   └── Layout.astro   # <head>, SEO/OG tags, fonts
├── pages/
│   ├── index.astro    # the whole marketing page
│   ├── quote-sent.astro   # no-JS success page
│   ├── quote-error.astro  # no-JS failure page
│   ├── robots.txt.ts
│   └── api/quote.ts   # the form endpoint (the only Worker route)
└── styles/
    ├── tokens.css  # mirrors tokens/*.css from the design project
    └── global.css
```

`src/components/` and `src/styles/tokens.css` are ports of the Claude Design
project. If you change how a component *looks*, change it there too, or the two
will drift.

## Brand assets

`public/assets/logo-cube.png` is cropped from the master logo lockup, and
`public/favicon.svg` is a simplified redraw of that cube — the real wireframe is
illegible below ~64px. `public/og.png` is generated by `npm run og`.

The brand source material (guidelines, the master logo, sticker artwork) is
deliberately **not** in this repo — it isn't needed to build or run the site, and
this repo is public. It lives alongside the project locally and is listed in
`.gitignore`.
